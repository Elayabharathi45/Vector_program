9-OS booting process
	1. Power-On Self Test (POST)
		When the computer is powered on, the BIOS (Basic Input/Output System) or UEFI (Unified Extensible Firmware Interface) firmware performs a POST to check the hardware components and ensure they are functioning correctly.
		Tasks: Checks memory, CPU, storage devices, and other peripherals.
	2. Bootloader Initialization
		After the POST, the BIOS/UEFI looks for the bootloader in the bootable storage device (e.g. hard drive. SSD)
		Tasks: Loads the bootloader into memory. The bootloader is responsible for loading the OS kernel.
	3. Bootloader Execution
		The bootloader (e.g. GRUB for Linux & Windows Boot Manager for Windows)takes control and loads the OS kernel into memory Tasks Displays a boot menu (if multiple OS's are installed), Loads the selected OS kernel and passes control to it.
	4. Kernel Initialization
		The OS kerel initializes and takes control of the system.
		Tasks: Sets up memory management.
			Initializes hardware components and device drivers.
			Starts essential kernel services.
	5. Init Process
		The kernel starts the initial process, often referred to as init (or systemd in modern Linux distributions).
		Tasks: Initializes system services and daemons.
			Sets up the user environment.
	6. User Space Initialization
		The init process starts user-space programs and services.
		Tasks: Starts login managers and graphical interfaces.
			Loads user-specific configurations.
	7. Login Prompt
		The system presents a login prompt or graphical login screen.
		Tasks: Allows the user to log in and start using the system.

	1.Independent Process
	2.Co-operating Process

	- fd[0] refers to read end of pipe, fd[1] refers to write end of file.
	- Data written to write end of pipe is buffered by kernel until it is read from read end of pipe.
	- pipe(2) 2nd arg - causes write system can fail once pipe full.
  ii)FIFO - size:0 
	- one way communication
	-  it is a special file similar to pipe, expect that it is accessed as a part of file system.

 	  without write to filesystem.

	- FIFO block until other end is opened also.
  iii)Message queues

44-Deadlock Situation
	- A situation where set of processes are blocked, as each process is holding resources and waits to acquire resources held by another process.

45-Thread

	- Each thread has its own: 
		Program counter, Registers, Stack and State.
	- Multiple threads running in a process share: 
		Address space, Heap, Static data, Code segments, File descriptors, Global variables, Child processes, Pending alarms, Signals and signal handlers.

46-Mutex
	- A type of mutual exclusion lock that ensures only one thread can access a shared resource at a time
	- It is uses to prevent race conditions by allow one thread to lock resource and others to wait until its unlocked
	- Used to protect shared resources from being simultaneously accessed by multiple threads
	- A mutex allows only one thread to access a resource at a time.
	- If a second thread tries to lock the mutex while another thread has it locked, it will be put to sleep until the mutex is unlocked

55-CPU scheduling?
	- It is a process that is used to improve efficiency by utilizing maximum CPU and providing minimum waiting time to tasks. 
	- CPU Scheduling is a process that allows one process to use the CPU while another process is delayed.
    (in standby) due to unavailability of any resources such as I/O, thus making full use of CPU.
	- The purpose of CPU Scheduling is to make the system more efficient and faster 
    Examples of scheduling algorithms are:
	1. First Come First Serve (FCFS)
	2. Shortest Job First (SJF)
	3. Round-Robin Scheduling
	4. Priority-based scheduling 
	5. Priority Scheduling

56- Preemptive Scheduling:
	- The OS allocates the resources to a process for a fixed amount of time. 
	- This switching happens because CPU may give other processes priority and substitute the currently active process for higher priority process.
	- During resource allocation, the process switches from running state to ready state or from waiting state to ready state.
    Non- Preemptive Scheduling:
	- The resource cant be taken from a process until the process completes execution. 
	- The switching of resources occurs when the running process terminates and moves to a waiting state.
	- This ensures that time-critical tasks can be executed promptly. 

57- Scheduling Policies
    1. First Come First Serve(FCFS) - (Non preemptive)
	- Tasks are execute on First come first serve basics (Queue)
	- Poor in performance as average wait time is high

    2. Shortest Job First - (Non preemptive)
	- First approach to minimize waiting time
	- The processor should knows in advance how much time processor will take

    3. Priority Scheduling - (Non preemptive & preemptive)
	- Each task is assigned a priority level
	- Task with highest priority will be given opportunity to use CPU
	- In preemptive version of algorithm, running task can be interrupted if a higher priority task ready to run state
	- In Non preemptive scheduling, once task is started it can't be interrupted by higher priority task
	- Task with same priority are executed as first come first serve or Round robin scheduling 

    4. Shortest Remaining time - (preemptive)
	- (Earliest deadline first)
	- It is dynamic-priority driven algorithm, in which highest priority is assigned to request that has earliest deadline 
	- A higher priority tasks always preempts lower priority tasks

    5. Round-Robin Scheduling - (preemptive)
	- There are no priority assigned to tasks
	- Each task is put into running state for fixed predefined time, this time is commonly referred as time-slice(quantum)
	- A task cannot run longer than the slice time
	-If a task is not completed by end of dedicated time-slice, it is interrupted, so next task from queue can run in following time slice
	-No starvation, no guarantee to meet deadlines 

58-Race Condition
	- Race conditions occur when multiple processes or threads access shared resources simultaneously, leading to unpredictable behavior.
	- This is a common issue in embedded systems, especially when dealing with interrupts or multitasking

59-What is starvation and aging in OS?
	When we use Priority Scheduling or Shortest Job First Scheduling, Starvation can happen, This algorithm is mostly used in CPU schedulers

	Starvation: 
	1. A problem that occurs when process not able to get required resources it needs for progress with its execution for long period of time. 
	2. low priority processes get blocked and high priority processes proceed towards completion because of which low priority processes suffer from lack of resources. 

	Aging: 
	1. It is a technique that is used to overcome the situation or problem of starvation. 
	2. It simply increases the priority of processes that wait in the system for resources for a long period of time.
	3. It is considered the best technique to resolve problem of starvation it adds an aging factor to the priority of each and every request by various processes for resources. 
	4. It also ensures that low-level queue jobs or processes complete their execution.

Memory Management
1. Memory partitioning 
	It is a technique used in operating systems to divide memory into sections for efficient allocation and management
		1. Fixed Partitioning
			Predefined memory blocks allocated at system startup.
			Processes are assigned to partitions based on their size.
			Limitation: Can lead to internal fragmentation if a process doesn’t use the full partition.
		2. Dynamic Partitioning
			Memory is divided based on the process's exact requirements.
			Reduces internal fragmentation but may lead to external fragmentation over time.
			Solution: Compaction—rearranging memory to create larger free blocks.
		3. Paging
			Divides memory into fixed-size pages mapped to frames in physical memory.
			Eliminates fragmentation issues and simplifies allocation.
			Used in virtual memory systems.
		4. Segmentation
			It is generally a memory management technique that divides processes into modules and parts of different sizes.
			These parts and modules are known as segments that can be allocated to process. 
			Can be combined with paging for segmented paging.
			Memory is divided into segments like code, data, stack and heap helping in organizing memory more efficiently

2. Paging 
	- When process ready for execution it is first loaded into virtual memory where the process divided into small units called pages
	- Pages are loaded from virtual memory to physical memory during execution
	- RAM is divided into small units called frames

3. Demand page
	- In virtual memory, it is type of swapping, pages of data are not copied from disk to RAM until they needed.
	- pages of a process are loaded into RAM only when they are accessed by the CPU, rather than all pages being loaded at once.
	- This is achieved through the Memory Management Unit (MMU) and operating system

4. Page fault
	- It's an exception or error that occurs when running program attempts to access memory page (fixed-size block of memory) that is not currently loaded in physical memory(RAM).
	- A invalid page fault or page fault error occurs when OS cannot find data in virtual memory.

5. Dirty page
	- If page has been modified, OS must preserve contents of that page so that it can be accessed at a later time.

6. Page Tables
	- Maintain the mapping between virtual and physical addresses.
	- Multi-Level Page Tables: Organize address translations more efficiently, reducing memory overhead.

7. Partition
	dividing the memory space into distinct regions or blocks, each with its own characteristics and access permissions
	1. Fixed  
	2. Dynamic(Variable)

8. Internal fragmentation 
	- It refers to wasted memory within a memory block that has been allocated to a process or task
	- If 1 process allocates 40MB memory but it uses only 4MB, other memory are wasted so it is called internal fragmentation 

9. External fragmentation
	- As processes are loaded, unloaded, and reloaded, memory is dynamically allocated and freed. 
	- This process can create "holes" or gaps in memory between allocated blocks

10. Compaction
	- It is a technique to overcome external fragmentation
	- It is the process in which free spaces are collected in the large memory chunk to make some space available for processes

11. Thrashing 
	- It refers to situation where system spends majority of its time of swapping data between RAM and secondary storage(usually the disk), instead of executing actual processes.

12. Virtual Memory
	- It acts as temporary storage that can be used along with RAM for computer processes. 
	- It is simply space where greater number of programs can be stored by themselves in the form of pages. 
	- It enables us to increase the use of physical memory by using a disk and also allows us to have memory protection.
	- It can be managed in two common ways by OS i.e., paging and segmentation. 

13. Purpose of virtual memory
 	Virtual memory allows execution of processes larger than physical memory by using disk space as an extension of RAM.
