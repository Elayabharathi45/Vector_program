7) Namespace
	- It is a directive that allows you to use all standard library components without explicitly qualifying them with std::
	- It is a declarative region used to localize global identifier names to avoid name collisions
	- A namespace creates a memory in which various program elements can be placed
	- Elements can be variables, functions, classes and templates
	- In C++ standard library, where all classes, functions and templates are declared in the namespace std;

8) Ambiguity error     
	- compiler cannot determine which function, variable, or class member to use due to multiple possibilities with the same name
	- This typically happens due to multiple inheritance, overloaded functions, or namespace conflicts
		1. When two base classes have the same member function, the derived class inherits both, causing ambiguity
		2. When two overloaded functions have parameters that lead to causing ambiguity
		3. If multiple namespaces define the same name, ambiguity occurs

9) default argument

10) inline function

23) static data member
	1. It is like global variable for its class, It is available to all objects of that class
	2. Default Value of static data members is 0
	3. Just one copy of this data members is created for entire class, since it is shared by all objects of that class
	4. A static data-member should not be initialized with constructor
	5. A static data-member is possible to access in 2 ways.
		i) object and dot operator
       	       ii) class name::
	6. non-static data member must be initialized only through constructor
	7. non-static data members must be accessed only through object and dot operator because it is a part of an object only

24) static member function

	2. A static member function can be involved by using the class name (instead of its objects) as follows:
              class_name :: function_name;
	3. Static member functions don't have an implicit "this" pointer as first parameter

28) Function Overriding
	- It occurs in inheritance when a derived class provides a new implementation for function that is already defined in base class
	- The mechanism of providing same function name with same arguments in a base class as well as derived class 

29) Function Binding
	The process of converting variables and functions into addresses is known as binding.
	Resolving a function call with it's proper function definition is called as function binding
		1. Compile time function binding
			- Also known as static binding.
			- The function call is resolved at compile-time.
			- Used for normal functions and function overloading
		2. Runtime function binding (same as virtual function)
			- Also known as dynamic binding.
			- Happens at runtime, enabling polymorphism.
			- Used when functions are marked as virtual in a base class.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    

30) Virtual function
	- It is used to enable runtime polymorphism, which means that correct function is called based on actual object type, not just the reference type.
	- A pointer of base class type (Base*) is used to refer an object of derived class type (Derived) --> Base* obj=new Derived();	
	- This is particularly useful in inheritance, where a derived class can override a base class function
	- It is a function, which must be present in base class, which may be overridden in derived class and informs the compiler to perform runtime polymorphism

31) Virtual table (vtable) 
	- It's a mechanism that supports runtime polymorphism by enabling dynamic binding of function calls. 
	- It's an internal lookup table, which virtual function should be called when using a base class pointer.
	- When a class contains virtual functions, compiler generates a vtable for the class.
	- Each object of the class stores a pointer to the vtable (known as a vptr).
	- When a virtual function is called using a base class pointer, vtable is used to dynamically resolve function from derived class at runtime.

31) pure virtual function
	A virtual function that is declared 0 but not defined in a base class 
		virtual <return-type> <function_name>(arg...) = 0;

32) Abstract class
	- If a class contains at least one pure virtual function 
        - If a virtual function declared as pure, it must be redefined in each derived class, else compilation of program is unsuccessful
	- A class having pure virtual function cannot be used to instantiate object of its own because of 0 address in virtual fun
		B *p; ->valid
		B b; -->error

33) Virtual Destructor
	- This is essential in polymorphism, when base class pointer holds dynamically created derived class object address

37) Singleton object in C++
	- It refers to design pattern that ensures a class has only one instance(object) and provides a global point of access to it.
	- This pattern is useful when you need centralized control, such as managing configurations, logging, or database connections.
	USES: 	Resource Management: Ensures a single access point for shared resources.
		Memory Optimization: Avoids unnecessary object creation.

38) Standard Template Library (STL)
	- It is a collection of template-based classes and functions that provide efficient implementations for common data structures and algorithms such as lists, stacks, arrays, sorting, searching
	- It is a generalized library we can use it with every data type without repeating code implementation 
	Types:
		1.Containers - used to store collections of elements efficiently.
			1) Sequence Containers
			vector – Dynamic array (fast access, efficient insertions at the end)
			deque – Double-ended queue (fast insertion/removal from both ends)
			list – Doubly linked list (efficient insertions/removals anywhere)

			2) Associative Containers
			set – Stores unique elements, sorted order
			map – Key-value pair storage (sorted by key)
			multiset – Like set, but allows duplicates
			multimap – Like map, but allows duplicate keys

			3) Unordered Containers
			unordered set, unordered map – Same as set/map but unsorted, based on hash tables.

		2.Algorithms - Predefined functions for operations like sorting and searching
		3.Iterators  - Tools to traverse elements in containers
		4.Functors(Functional objects)   - These are objects that act like functions and can be passed to STL algorithms

39) Smart pointers

	- help to prevent memory leaks and eliminate the need for manual delete operations.
