1. What is __init__
	- It is a constructor method in Python and is automatically called to allocate memory when a new object/instance is created.
	- All classes have a __init__ method associated with them.
	- It helps in distinguishing methods and attributes of a class from local variables.

2. List
	- Lists in python contain elements of different data types and it is mutable.
	- lists can be modified, appended or sliced 
	- It has the disadvantage of consuming large memory
	- Lists are represented with square brackets ['sara', 6, 0.19]
  Tuple
	- Tuples are sequence data types that can store a collection of objects in Python, it is immutable
	- tuples remain constant and cannot be modified
	- tuples are represented with parenthesis ('ansh', 5, 0.97)
  Dictionary
	- Dict stores values in key : value pairs
	- value in dict can be any datatype and can be duplicated
	- values can be changeable but keys can't be repeated and immutable

3. Slicing ()
	- As the name suggests, ‘slicing’ is taking parts of.
	- Syntax for slicing is [start : stop : step]
	- start is the starting index from where to slice a list or tuple
	- stop is the ending index or where to stop.
	- step is the number of steps to jump.
	- Default value for start is 0, stop is number of items, step is 1.
	- Slicing can be done on strings, arrays, lists and tuples.

4. Docstring 
	- Documentation string or docstring is a multiline string used to document a specific code segment.
	- The docstring should describe what the function or method does, making code readable & maintainable.

5. Self parameter
	- Self is used to represent the instance of the class.
	- With this keyword, you can access the attributes and methods of the class in python. 

6. Modules and packages
	A module is simply Python file (.py) contains functions, classes, variables you can reuse in other programs. 
	A package is a collection of modules stored in directory with a special file __init__.py. This allows Python to treat the directory as package.

7. Pass keyword
	- keyword represents a null operation in Python, just a placeholder
	- used for filling empty blocks of code which execute during runtime but has yet to be written.
	- useful in scenarios where you define function, class, or loop but don’t want to add actual code yet

8. Static typed language - Data Types are checked before execution.
   Dynamic typed language - Data Types are checked during execution.
	Python is an interpreted language, executes each statement line by line and thus type-checking is done on the fly, during execution. Hence, Python is a Dynamically Typed Language.

9. Decorators
	- decorators are functions that take another function as input and extend its functionality without modifying its source code
	- decorator is applied to a function using @decorator_name syntax
	- used to modify or extend the behavior of functions and methods without changing their original code

10.scope resolution
	- objects within the same scope have same name but function differently.
	- Local (L): The innermost scope, where variables are defined inside a function
	- Enclosing (E): The scope of any enclosing function (used in nested functions)
	- Global (G): The module-level scope (variables defined at the top level of a script)
	- Built-in (B): The standard library scope, where built-in functions like print() and len() exist

11. Lambda function
	- Lambda is an anonymous function in Python, that can accept any number of arguments, but can only have a single expression.
	- It is generally used in situations requiring an anonymous function for a short time period
	It can be used in two ways
	1. Assigning lambda functions to a variable:
		mul = lambda a, b : a * b
		print(mul(2, 5))    # output => 10
	2. Wrapping lambda functions inside another function:	
		def myWrapper(n):
		 return lambda a : a * n
		mulFive = myWrapper(5)
		print(mulFive(2))    # output => 10

12. Negative indexes
	- Negative indexes are indexes from end of list or tuple or string.
	- Arr[-1] means the last element of array Arr[]

13. *args
	*argv in function definitions is used to pass a variable number of arguments to a function
		def fun(*argv):
    			for arg in argv:
        		print(arg)
		fun('Hello', 'Welcome', 'to', 'GeeksforGeeks')
    **kwargs 
 	**kwargs in function definitions is used to pass a variable length argument list to a function
		def fun(**kwargs):
    			for k, val in kwargs.items():
       			print("%s == %s" % (k, val))		
		fun(s1='Geeks', s2='for', s3='Geeks')
	
14. Set 
	- a set is a built-in data type that represents an unordered collection of unique elements
	- It is iterable, mutable and has no duplicate elements
	my_set = {1, 2, 3}
	my_set.add(4)		 # Result: {1, 2, 3, 4}
	my_set.remove(2)	 # Result: {1, 3, 4}
	my_set.discard(5)	 # Result: {1, 3, 4}

	set1 = {1, 2, 3}
	set2 = {3, 4, 5}
	union_set = set1 | set2		# Result: {1, 2, 3, 4, 5}
	intersection_set = set1 & set2  # Result: {3}
	difference_set = set1 - set2    # Result: {1, 2}
